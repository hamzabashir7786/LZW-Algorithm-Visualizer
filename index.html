Algorithm Theory

Understanding LZW from first principles.

🧠

What is LZW?

Lempel–Ziv–Welch (LZW) is a universal lossless data compression algorithm published by Abraham Lempel, Jacob Ziv, and Terry Welch in 1984. It builds a dictionary of substrings dynamically during encoding — no prior knowledge of the data is needed.

Unlike Huffman coding (which assigns shorter codes to frequent symbols), LZW replaces variable-length strings with fixed-length codes, achieving compression when repeated patterns exist.

⚙️

How It Works (Step-by-Step)

  1. Initialize the dictionary with all 256 ASCII characters (codes 0–255).
  2. Read input character C. Set current string W = C.
  3. While W+C exists in the dictionary: append C to W, read next C.
  4. Output the code for W.
  5. Add W+C to the dictionary with next available code.
  6. Set W = C. Repeat from step 3 until input is exhausted.
  7. Output the code for the final W.
🌐

Real-World Applications

  • ZIP Early ZIP and Unix compress utility used LZW
  • GIF GIF image format uses LZW for pixel data
  • TIFF TIFF image compression option
  • PDF PDF files support LZW stream compression
  • NET Network protocol data compression
⚖️

Advantages & Limitations

✓ Advantages
  • Single-pass algorithm — O(n)
  • No prior training data needed
  • Adaptive to input patterns
  • Decompression also O(n)
✗ Limitations
  • Dictionary can grow large
  • Poor on already-compressed data
  • Patent issues historically (GIF)
  • Superseded by LZ77/Deflate in practice
📐

Algorithm Pseudocode

function LZW_Compress(input):
    // Step 1: Initialize dictionary with ASCII chars
    dict ← {char: code  for code, char in enumerate(ASCII_256)}
    next_code ← 256
    output   ← []

    W ← input[0]                      // current string

    for i ← 1 to len(input) - 1:
        C ← input[i]                   // current character
        if W + C in dict:
            W ← W + C                  // extend match
        else:
            output.append(dict[W])     // emit code for W
            dict[W + C] ← next_code    // add W+C to dict
            next_code ← next_code + 1
            W ← C                      // reset W to C

    output.append(dict[W])             // emit final code
    return output
